1.에러 로그가 없으면 뭐가 아려울까?
언제,얼마나 자주,발생했는지 모르고 특정 사용자만 그런건지도 몰라서 재현이 어렵다 즉, 추측으로 대응하게 되는데 이게 가장 위험하다.

2.개발자/운영/보안 입장에서 왜 중요할까?
개발자 - 코드 어느부분에서 터졌는지 확인
       -재현-수정-재배포
운영- 시스템 장애인지 서버 과부하인지 네트워크 문제인지 판단
보안- 공격 시도인지 단순 실수인지 구분
    -특정 IP에서 반복 시도있는지 확인
    -비정상 패턴 탐지
보안은 이게 공격인지 아닌지 판단하기 위해 로그를 본다.

3.에러 메세지가 너무 자세하면 왜 위험할까?
로그인 실패 시
"아이디가 존재하지 않습니다." VS "아이디 또는 비밀번호가 틀렸습니다." 중에 두번때가 더 안전한 이유
공격자가 아이디가 존재하지 않다고 뜨면 버리고 비밀번호 틀렸다고 뜨는 아이디만 모아서 비밀번호만 집중 공격함
아이디 또는 비밀번호가 틀렸다고 뜨면 아이디가 틀린건지 비밀번호가 틀렸는지 구분 못함 그래서 공격자가 정보를 못 얻음
